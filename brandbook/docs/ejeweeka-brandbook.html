Who we are

ejeweeka is a daily wellness intelligence app — tracking nutrition, sleep, activity, habits, and daily wellbeing. Premium mass positioning: an exclusive club feeling, without cold elitism.

The primary mark is the "eje" sign: three letters that form a playful face — two "e"s as eyes, "j" as the center element, and the arc as a smile. Neon violet-magenta glow on a deep dark background.

Status System

Access levels are called Status, not "subscription", "plan", or "tier". Compliance-aware wording — reduces App Store review risk.

Terminology rule: Never use "subscription", "price", "buy", "tariff", "купить", "цена", "тариф", "подписка" in any UI copy. Use: "статус", "открыть статус", "активировать статус", "возможности статуса". This approach reduces moderation risk — it does not guarantee App Store approval.

Logo Usage

Rules & Do/Don'ts

✓ Do

✓
Use app icon master on dark / OLED backgrounds with full glow
✓
Use transparent mark on dark backgrounds (site, splash, presentations)
✓
Scale proportionally — width AND height together
✓
Use tiny 29–40px favicon-optimized PNG for small UI placements
✓
Use monochrome mark (app-icon desaturated) for documents and email
✓
Maintain clear space of at least 0.5× icon height on all sides
✓
Write ejeweeka — always lowercase, everywhere

✗ Don't

✗
Use capital letters: Ejeweeka, EJEWEEKA, EjeWeeka
✗
Stretch, compress, or rotate the mark
✗
Change the smile shape, letter spacing, or eje proportions
✗
Place the neon mark on white/light backgrounds without proper masking — glow looks dirty
✗
Use the complex neon logo at 16px — use tiny favicon-optimized version instead
✗
Add medical symbols: cross, heart, leaf, scale, fork, spoon, pulse, pill
✗
Make the entire wordmark a gradient — gradient on the mark only
✗
Place logo on noisy, photographic, or pattern-heavy backgrounds
✗
Use cheap drop shadows — use glow or no shadow

When to use which version

App Icon Master
App Store, Google Play, social avatar, PWA icon, Telegram, email avatar, any platform expecting a square icon.
Transparent Neon Mark
Website header/hero, splash screen, onboarding, presentations on dark backgrounds. Refined mask required for light backgrounds.
Tiny Mark (favicon)
Browser tab (16/32/48px), app notification badges, small UI contexts where full detail can't render. Use favicon-NN.png files.
Horizontal Lockup
Website nav header, in-app navigation bar (if logo is shown), partner placements, presentation headers.
Vertical Lockup
Splash screen, welcome screen, onboarding first slide, presentation title slide. Center-aligned, OLED background.
Monochrome (desaturated icon)
PDF documents, invoices, email signatures (non-HTML), printed materials. Never the neon version on white paper.
