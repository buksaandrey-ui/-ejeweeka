# ⛔ ПЛАТЁЖНЫЕ ОГРАНИЧЕНИЯ — РОССИЯ, МАЙ 2026

> **Дата фиксации**: 2026-05-13
> **Статус**: ДЕЙСТВУЕТ. Учитывать ВО ВСЕХ решениях.

---

## 1. IAP (In-App Purchase) — НЕ РАБОТАЕТ ДЛЯ РФ МАССОВОГО РЫНКА

- Apple IAP **неработоспособен** для большинства российских пользователей.
- Последние способы пополнения баланса Apple ID (МТС, Билайн) **отключены**.
- Google Play Billing — аналогично ограничен.
- **Вывод для РФ**: IAP не является массовым каналом монетизации для РФ-аудитории.
- **Вывод для Global**: IAP остаётся основным каналом для международных пользователей и для Apple App Store compliance.

## 2. Telegram — УБРАН ИЗ КРИТИЧЕСКОГО ПУТИ

- На май 2026 года Telegram испытывает проблемы с доступностью в РФ.
- Telegram **удалён из MVP как критический канал** (оплата, восстановление, инвайты).
- Telegram может использоваться ТОЛЬКО как **optional off-app marketing/support channel** на web, но НЕ внутри iOS app.

## 3. Архитектура монетизации — DUAL CHANNEL

| Канал | Аудитория | Статус | Роль |
|---|---|---|---|
| **Apple IAP (StoreKit)** | iOS / Global | ✅ Работает | Основной канал для iOS app |
| **Web ЮKassa (ejeweeka.ru)** | РФ | ✅ Работает | Основной канал для РФ-аудитории |
| **Backend Entitlement** | Все | ✅ Source of Truth | Единая система статусов |
| IAP для РФ | РФ | ⚠️ Ограничен | Работает для тех, кто может пополнить Apple ID |
| Telegram-бот | — | ❌ Убран из MVP | Optional off-app support only |

## 4. Архитектурные следствия

- **iOS app**: Монетизация ТОЛЬКО через Apple IAP. Никаких ссылок на web checkout.
- **Web РФ**: Оплата через ЮKassa (СБП, карты РФ). Промокоды — только на web.
- **Status sync**: Пользователи, купившие статус на web, входят в app по email и синхронизируют статус.
- **Восстановление**: iOS = Restore Purchases; Web = восстановление через email на ejeweeka.ru.

## 5. ЗАПРЕЩЕНО в iOS-приложении

- Ссылки на web checkout / /subscribe
- Упоминания ЮKassa, СБП, банковских карт
- Telegram-оплата, Telegram-бот
- Промокоды, скидки
- Deep links из app на web checkout
- Коды активации (activation codes)
- Любые CTA, направляющие на внешнюю оплату

---

> ⚠️ **ПРАВИЛО**: The app must not link to web checkout. Russian users may purchase on web only through off-app channels (organic search, marketing, support).
> При проектировании ЛЮБОГО нового flow: "Содержит ли это запрещённую строку или CTA на web-оплату?" Если ДА — решение неприемлемо.
