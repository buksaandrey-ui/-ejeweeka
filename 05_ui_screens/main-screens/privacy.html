Политика конфиденциальности

Дата вступления в силу: 23 апреля 2026 г.

Настоящая Политика конфиденциальности описывает, как мобильное приложение Health Code (далее — «Приложение») собирает, использует и защищает вашу информацию.

Главное: Health Code использует архитектуру «Zero-Knowledge». Ваши медицинские данные хранятся ТОЛЬКО на вашем устройстве. Мы не имеем доступа к вашему профилю здоровья.

1. Какие данные мы собираем

Данные на устройстве (не передаются на сервер):

  • Ваш профиль: возраст, пол, рост, вес, цели питания
  • Медицинские данные: заболевания, аллергии, анализы крови
  • Пищевые предпочтения и ограничения
  • Дневник питания, трекер воды, трекер веса

Данные, передаваемые на сервер (анонимно):

  • Анонимный идентификатор устройства (UUID) — для привязки статуса
  • Обобщённый запрос к AI (без персональных данных) — для генерации плана питания
  • Фотографии еды (только при использовании функции «Анализ фото») — обрабатываются API Google Gemini и сразу удаляются

2. Как мы используем данные

  • Генерация плана питания: ваш профиль (возраст, вес, цели) используется локально для формирования запроса к AI. Сам запрос содержит только обобщённые параметры.
  • Фото-анализ: фотография отправляется в Google Gemini API для распознавания еды и расчёта калорий. Фото не сохраняется на наших серверах.
  • Статус: оформляется через Telegram-бот (@healthcode_bot) и ЮKassa. Мы не храним данные платёжных карт — это делает ЮKassa напрямую.

3. Хранение данных

  • На устройстве: все персональные данные хранятся в localStorage вашего устройства. При удалении приложения данные стираются.
  • На сервере: мы храним только анонимный UUID и статус доступа. Медицинские данные на сервер не передаются.

4. Третьи стороны

  • Google Gemini API — для генерации планов питания и анализа фото. Условия использования.
  • ЮKassa — платёжный шлюз для оформления статусов. Политика конфиденциальности.
  • Supabase — для хранения базы медицинских знаний (не персональных данных). Политика конфиденциальности.

5. Права пользователя

  • Удаление данных: удалите приложение — все локальные данные удалятся автоматически.
  • Запрос на удаление серверных данных: напишите на support@aidiet.ru — мы удалим ваш анонимный UUID в течение 48 часов.
  • Отказ от аналитики: приложение не использует трекеры (Facebook Pixel, Google Analytics и т.п.) в текущей версии.

6. Безопасность

Все API-запросы передаются по зашифрованному каналу (HTTPS/TLS 1.3). Серверы размещены в ЕС (Франкфурт, Германия) на платформе Render.

7. Дети

Приложение не предназначено для лиц младше 16 лет. Мы не собираем данные детей намеренно.

8. Медицинский дисклеймер

Health Code не является медицинским устройством и не предоставляет медицинских рекомендаций. Информация в приложении носит исключительно ознакомительный характер. Перед изменением рациона проконсультируйтесь с лечащим врачом.

9. Изменения

Мы можем обновлять настоящую Политику. Актуальная версия всегда доступна по адресу aidiet.ru/privacy.

10. Контакты

По вопросам конфиденциальности: support@aidiet.ru
